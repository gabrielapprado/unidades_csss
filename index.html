Unidades no CSS

As unidades são formas de definir tamanhos em um site. Algumas unidades são fixas, já outras são capazes de se adaptar a tela.

Unidades absolutas

As unidades absolutas são fixas e não se adaptam a tela.

px (pixel)

O pixel é a unidade virtual padronizda pelo navegador, 1px = 1/96 de 1 polegada (in). Quando usamos pixels isso implica que queremos que um elemento ocupe determinado espaço da tela

      p {
        font-size: 20px;
      }

      .caixa {
        width: 300px;
        height: 40px;
      }

Isso significa que todos os elementos p (parágrafos) da página vão ter um tamanho que o navegador considera equivalente a 20 unidades de tela.

Resultado visual

Este parágrafo tem font-size: 20px

300px

Quando usar:

Bordas, margens, ícones pequenos, ajustes finos - porque o pixel está associado a elementos de um tamanho específico e não devem variar com base no tamanho da tela, elementos que representam um tamanho visual fixo.

Quando não usar:

Layouts muito responsivos e em tipografia acessível- porque esses cenários dependem da adaptação do elemento ao caso de cada usuário, como aumentar a fonte

cm, mm, in, pt e pc (Unidades físicas)

Elas são chamadas de unidades físicas porque são definidas com base em medidas do mundo real, independentes de qualquer dispositivo digital.

No CSS, todas elas são definidas a partir de uma unidade principal: a polegada (in).

1in = 2.54cm = 25.4mm = 72pt = 6pc = 96px

O navegador utiliza essa conversão para calcular o layout

Quando usar:

O meio de saída for conhecido, como a impressão, normalmente são utilizadas para PDFs, documentos, jornais, etiquetas, entre outros.

Quando não usar:

As unidades físicas não são adequadas para interfaces web, pois a renderização depende do dispositivo e da densidade de pixels (DPI), tornando essas medidas inconsistentes na prática.

Unidades Relativas

As unidades relativas não possuem tamanho fixo, mas seu tamanho dependende da relação com outra propriedade de comprimento(elemento pai, tela, html).

% (porcentagem)

Elemento pai e filho

        <div class="pai">
          <p class="filho">Eu sou o elemento filho</p>
        </div>

É um valor baseado no tamanho do elemento pai.Se o elemento pai tem 400px e o filho 50%, significa que o elemento filho tem 200px.

        .container {
          width: 100%;  /* ocupa toda a largura do pai */
        }

        .filho {
          width: 50%;   /* metade do container */
        }

Resultado visual

filho — 50%

Quando usar:

A porcentagem é indicada principalmente para divisão de espaços ou alinhamento de elementos.

Quando não usar:

É recomendado evitar a porcentagem quando os elementos pai são mal definidos, pois pode assumir um tamanho desproporcional na página.

em

Depende do tamanho da fonte do elemento pai.

      .pai {
        font-size: 20px;
      }

      .filho {
        font-size: 2em;  /* 2 × 20px = 40px */
      }

Resultado visual

pai — font-size: 20px filho — 2em = 40px

Isso significa que se o elemento pai tem um tamanho de fonte de 20px, se o seu elemento filho tem um tamanho de 2em o tamanho da fonte é 2x20= 40px.

Quando usar:

Situações que exigem proporcionalidade, como espaçamento que acompanha o texto ou elementos que precisam aumentar junto com a fonte.

Quando não usar:

Elementos muito aninhados, o em vai acumulando e pode crescer demais.

rem

Diferente do em que é baseado no elemento pai, o rem depende do tamanho de fonte do elemento raiz (html).

Exemplo: se o html tem 16px de tamanho de fonte, um texto com 2rem terá 2x16= 32px. 

      html {
        font-size: 16px;  /* raiz */
      }

      .a { font-size: 1rem; }    /* 16px */
      .b { font-size: 1.5rem; }  /* 24px */
      .c { font-size: 2rem; }    /* 32px */

Resultado visual

1rem — 16px — Texto base
1.5rem — 24px — Subtítulo
2rem — 32px — Título

Quando usar:

Tamanho de texto e layout escalável.

Quando não usar:

rem não é indicado quando o elemento precisa ser proporcional ao elemento pai, pois ele usa como base apenas o tamanho global da página.

vw(viewport width)

Baseado na largura da tela

Exemplo: 1vw representa 1% da largura da tela.

      .banner {
          width: 50vw;   /* 50% da largura da tela */
      }

      h1 {
        font-size: 5vw;  /* tipografia responsiva */
      }

Resultado visual — barra ocupa 50% da largura disponível

CSS

Quando usar:

Layout que acompanha o tamanho da tela, seções que ocupam largura total e tipografia responsiva.

Quando não usar:

Elementos que precisam de limite, pois pode aumentar ou diminuir muito dependendo da tela.

vh(viewport height)

Baseado na altura da tela

Exemplo: 1vh representa 1% da altura da tela

      .secao {
        height: 100vh;  /* ocupa a tela inteira */
      }

      .modal {
        height: 80vh;
      }

Resultado visual — proporção em relação à altura da janela

Ocupa 80% do espaço

Quando usar:

Layouts flexíveis e seções que ocupam a altura total.

Quando não usar:

Em celulares é preciso ter um cuidado extra, uma vez que a altura da página muda e isso pode quebrar o layout.

vmin e vmax

O vmin usa o menor valor entre um intervalo, no caso de medidas o menor valor entre largura e altura

Enquanto o vmax utiliza o maior valor.

      .vmin {
        width: 20vmin;
        height: 20vmin;
        background: lightgreen;
      }

      .vmax {
        width: 20vmax;
        height: 20vmax;
        background: lightcoral;
      }
vmin
vmax

VMIN- utiliza o menor lado da tela

VMAX- utiliza o maior lado da tela

Quando usar:

Manter a proporção de um elemento independente da orientação do dispositivo, exemplo: quando o celular rotaciona.

Quando não usar:

Componentes internos ou layouts que dependem do elemento pai.
